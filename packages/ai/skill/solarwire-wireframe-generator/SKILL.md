# SolarWire 线框图生成器

## 配置项

- **输出目录名称**：`.solarwire`（如需修改，只需更改此处即可）

---

## ⚠️ 核心原则（必须严格遵守）

### 一、语法铁律

```
1. 所有元素必须有坐标 @(x,y)
2. 属性直接写，不用方括号：w=100 h=40（不是 [w=100 h=40]）
3. 文本内容必须用双引号："登录"（不是 登录）
4. 属性顺序：内容 → 坐标 → 尺寸 → 其他属性 → note
```

**正确示例：**
```solarwire
["登录"] @(100,50) w=100 h=40 bg=#3498db c=white note="提交登录表单"
"用户名" @(100,100)
```

**错误示例：**
```solarwire
["登录"]                    // ❌ 没有坐标
["登录"] [w=100 h=40]       // ❌ 属性用了方括号
["登录"] @(100,50) w=100    // ❌ 缺少高度
```

### 二、元素选择原则

**目标：线框图要干净、明确、贴近实际页面显示**

| 场景 | 推荐元素 | 示例 |
|------|---------|------|
| 按钮操作 | 矩形 `[]` | `["登录"] @(100,50) w=100 h=40 bg=#3498db c=white` |
| 卡片/容器 | 圆角矩形 `()` | `("用户信息卡片") @(100,50) w=300 h=200` |
| 头像/图标按钮 | 圆形 `(())` | `((头像)) @(100,50) w=40` |
| 标签/文字 | 纯文本 `""` | `"用户名" @(100,50)` |
| 输入框 | 矩形 + 占位符文字 | `["请输入用户名"] @(100,50) w=280 h=40 bg=#fff b=#ddd` |
| 图标（无真实图片时） | 占位符 `[?]` | `[?"搜索"] @(100,50) w=32 h=32` |
| 真实图片 | 图片 `<url>` | `<https://example.com/logo.png> @(100,50) w=40` |
| 分隔线 | 线条 `--` | `-- @(0,100)->(400,100) b=#eee` |
| 数据表格 | 表格 `##` | `## @(100,50) w=500 border=1` |

**禁止滥用：**
- ❌ 不要用占位符 `[?]` 表示按钮或文字（用矩形或纯文本）
- ❌ 不要用矩形 `[]` 表示纯标签（用纯文本 `""`）
- ❌ 不要用圆形 `(())` 表示非头像/图标按钮的元素

**图标处理原则：**
- 有真实 URL → 用 `<url>` 加载真实图片
- 无真实图片但需表示图标 → 用 `[?"图标名"]` 占位符
- 图标按钮 → 用矩形 `["图标名"]` 或圆形 `(("图标名"))`

### 三、页面组织规则

**每个 SolarWire 代码块只处理一个独立视图：**

| 情况 | 处理方式 | 示例 |
|------|---------|------|
| 弹窗/对话框 | 单独一个 SolarWire 片段 | `## 登录失败弹窗` + 独立代码块 |
| 页面不同状态 | 每个状态单独一个片段 | `## 登录页 - 加载中状态`、`## 登录页 - 错误状态` |
| 标签页切换 | 每个标签页单独一个片段 | `## 设置页 - 基本信息标签`、`## 设置页 - 安全设置标签` |

**禁止在一个代码块中混合多个视图状态。**

### 四、容器矩形要求

**每个页面必须有容器矩形：**

```solarwire
!title="页面名称"
!c=#333
!size=14
!bg=#f5f7fa
!r=8

// 容器矩形 - 代表屏幕/设备边界，放在最下层
[] @(0,0) w=375 h=812 bg=#fff

// 页面内容...
```

**容器矩形规范：**
- 放在代码块最前面
- 使用 `[]` 矩形（不写文字内容）
- `bg=#fff` 白色背景
- 尺寸根据场景：
  - 移动端：`w=375 h=812`（iPhone X）或 `w=390 h=844`（iPhone 12+）
  - Web 端：`w=1440 h=900` 或根据实际
  - 后台管理：`w=1920 h=1080`

**容器尺寸原则：容器必须包含所有子元素**

当内容超出屏幕/弹窗边界时，容器应该扩展以包含所有内容，而不是让子元素超出容器。

| 场景 | 正确做法 | 错误做法 |
|------|---------|---------|
| 弹窗内容多 | 弹窗容器高度扩展，超出屏幕范围 | 表单元素超出弹窗容器 |
| 表格行数多 | 表格容器高度扩展 | 表格行超出表格容器 |
| 表单字段多 | 表单容器高度扩展 | 输入框超出表单容器 |
| 列表项多 | 列表容器高度扩展 | 列表项超出列表容器 |

**示例：弹窗内容超出屏幕**
```solarwire
// 容器矩形 - 超出屏幕范围以容纳所有内容
[] @(0,0) w=1440 h=2000 bg=#fff

// 弹窗容器 - 高度足够容纳所有表单元素
("新增客户") @(420,100) w=600 h=1800 bg=white r=12

// 表单元素都在弹窗容器内
"客户姓名" @(460,150)
["请输入客户姓名"] @(460,175) w=520 h=40
// ... 更多表单元素，都在弹窗容器范围内
```

**禁止：子元素超出父容器边界。**

### 五、Note 写作规范

**目标：详细到开发可执行、测试可写用例**

**好的 Note：**
```solarwire
["登录"] @(100,50) w=100 h=40 note="点击后验证账号密码。
- 成功：跳转首页，保存登录状态
- 失败：显示「用户名或密码错误」，清空密码框
- 按钮禁用条件：用户名或密码为空
- 防重复：点击后按钮禁用3秒，或请求返回后恢复"

["用户名"] @(100,100) w=280 h=40 note="输入框，支持手机号或邮箱登录。
- 自动去除首尾空格
- 格式校验：手机号11位数字，邮箱需含@和域名
- 错误提示：格式错误时输入框变红，下方显示「请输入正确的手机号或邮箱」
- 最大长度：50字符"
```

**Note 必须包含（按需）：**
1. **功能行为**：点击/操作后发生什么
2. **成功/失败场景**：不同结果的处理方式
3. **输入限制**：格式、长度、必填等
4. **边界条件**：空值、超长、特殊字符等
5. **状态变化**：禁用条件、加载状态等
6. **错误提示**：什么情况显示什么提示

**❌ 禁止在 Note 中写的内容：**
- ❌ 视觉细节：颜色、字体、圆角、阴影、动画效果
- ❌ 技术实现：接口名、数据库字段、加密方式、框架名称
- ❌ 尺寸数值：宽度、高度、间距（线框图已展示）

**错误示例：**
```solarwire
// ❌ 包含视觉细节
["登录"] note="蓝色按钮，圆角8px，悬停时变深蓝，点击有缩放动画"

// ❌ 包含技术实现
["登录"] note="调用 /api/login 接口，POST 请求，返回 token 存入 localStorage"

// ❌ 包含尺寸数值
["登录"] note="宽度300px，高度48px，与输入框对齐"
```

### 六、开发就绪规范

**目标：PRD 生成后可直接进入开发阶段**

#### 1. 数据字段定义

**每个表单/列表必须定义字段：**

在页面说明中添加字段定义表格：

```markdown
### 数据字段定义

| 字段名 | 类型 | 必填 | 默认值 | 校验规则 | 说明 |
|-------|------|-----|-------|---------|------|
| username | string | 是 | - | 手机号11位或邮箱格式 | 用户名，支持手机号或邮箱登录 |
| password | string | 是 | - | 6-32字符，含字母和数字 | 密码 |
| rememberMe | boolean | 否 | false | - | 记住登录状态，7天免登录 |
```

#### 2. 业务规则说明

**每个页面必须说明业务规则：**

```markdown
### 业务规则

1. **登录规则**
   - 支持手机号或邮箱登录
   - 连续失败5次后锁定账号15分钟
   - 登录成功后跳转来源页面或首页

2. **权限规则**
   - 未登录用户：只能访问登录页、注册页
   - 普通用户：可查看、不可编辑删除
   - 管理员：全部权限

3. **数据规则**
   - 用户状态：启用/禁用，禁用用户无法登录
   - 角色类型：管理员、普通用户
   - 创建时间：系统自动生成，不可修改
```

#### 3. 状态流转说明

**复杂交互必须说明状态流转：**

```markdown
### 状态流转

**订单状态：**
- 待支付 → 已支付 → 待发货 → 已发货 → 已完成
- 待支付 → 已取消
- 已支付 → 申请退款 → 退款中 → 已退款
- 已发货 → 申请退货 → 退货中 → 已退货

**触发条件：**
- 用户支付成功：待支付 → 已支付
- 管理员发货：待发货 → 已发货
- 用户确认收货：已发货 → 已完成
```

#### 4. 异常处理说明

```markdown
### 异常处理

| 异常场景 | 处理方式 |
|---------|---------|
| 网络错误 | 显示「网络连接失败，请检查网络后重试」，提供重试按钮 |
| 服务端错误 | 显示「服务器繁忙，请稍后重试」，自动记录错误日志 |
| 权限不足 | 显示「您没有权限执行此操作」，跳转登录页 |
| 数据为空 | 显示空状态占位图 + 「暂无数据」提示 |
| 参数校验失败 | 输入框变红，下方显示具体错误提示 |
```

#### 5. 表格数据说明

**表格必须在 Note 中说明：**

```solarwire
## @(264,316) w=1632 border=1 note="用户列表表格。
- 数据来源：用户管理模块，按创建时间倒序排列
- 默认每页10条，支持分页（10/20/50/100）
- 支持按姓名、邮箱、手机号搜索
- 支持按状态筛选（全部/启用/禁用）
- 权限：管理员可编辑删除，普通用户只能查看
- 排序：支持按创建时间、姓名排序
- 批量操作：支持批量删除、批量修改状态"
```

### 文档级声明

```solarwire
!title="页面标题"
!c=#333           // 默认文字颜色
!size=14          // 默认字体大小
!bg=#f5f7fa       // 背景色
!r=8              // 默认圆角
```

### 基本元素

| 符号 | 用途 | 示例 |
|-----|------|-----|
| `[]` | 按钮、输入框、容器 | `["确定"] @(100,50) w=80 h=36` |
| `()` | 卡片、圆角容器 | `("提示卡片") @(100,50) w=200 h=100` |
| `(())` | 头像、圆形图标 | `((头像)) @(100,50) w=40` |
| `""` | 纯文本、标签 | `"用户名" @(100,50)` |
| `[?]` | 图标占位符 | `[?"搜索"] @(100,50) w=32 h=32` |
| `<url>` | 真实图片 | `<https://example.com/logo.png> @(100,50) w=40` |
| `--` | 分隔线 | `-- @(0,100)->(400,100)` |
| `##` | 表格容器 | `## @(100,50) w=500 border=1` |
| `#` | 表格行 | `# bg=#eee` |

### 表格元素详解

**表格结构：**
```solarwire
## @(x,y) w=宽度 border=边框宽度
  # bg=表头背景色
    "列1"
    "列2"
    ...
  #
    "数据1"
    "数据2"
    ...
  # bg=交替背景色（可选）
    ...
```

**完整示例：**
```solarwire
## @(264,316) w=1632 border=1
  # bg=#eee
    "ID"
    "用户姓名"
    "邮箱"
    "手机号"
    "角色"
    "状态"
    "创建时间"
    "操作"
  #
    "1"
    "张三"
    "zhangsan@example.com"
    "13800138001"
    "管理员"
    "启用" c=#4CAF50
    "2024-01-15"
    ["编辑"] note="打开编辑弹窗"
    ["删除"] bg=#f44336 c=white note="删除该用户"
  # bg=#f9f9f9
    "2"
    "李四"
    "lisi@example.com"
    "13800138002"
    "普通用户"
    "启用" c=#4CAF50
    "2024-01-16"
    ["编辑"] note="打开编辑弹窗"
    ["删除"] bg=#f44336 c=white note="删除该用户"
```

**表格 Note 规范：**
```solarwire
## @(264,316) w=1632 border=1 note="用户列表表格。
- 数据来源：用户管理模块，按创建时间倒序排列
- 默认每页10条，支持分页
- 支持按姓名、邮箱、手机号搜索
- 支持按状态筛选（全部/启用/禁用）
- 权限：管理员可编辑删除，普通用户只能查看
- 排序：支持按创建时间、姓名排序"
```

**表格行内操作按钮：**
- 放在最后一列
- 使用矩形 `[]` 表示按钮
- 多个按钮水平排列，间距 10px

**表格内元素 Note 支持：**
- 表格内的所有元素（包括表头、数据行、操作按钮）都可以使用 `note` 属性
- 每个元素的 note 都会被单独收集和显示
- note 编号按元素出现顺序自动分配
- 示例：
```solarwire
## @(264,316) w=1632 border=1 note="用户列表表格"
  # bg=#eee
    "ID" note="用户唯一标识符，自增整数"
    "用户姓名" note="显示用户全名"
    ...
  #
    "1"
    "张三"
    ...
    ["编辑"] note="打开编辑用户弹窗"
    ["删除"] bg=#f44336 c=white note="删除该用户"
```

### 常用属性

| 属性 | 说明 | 示例 |
|-----|------|-----|
| `w` `h` | 宽度、高度 | `w=100 h=40` |
| `bg` | 背景色 | `bg=#3498db` |
| `c` | 文字颜色 | `c=white` 或 `c=#fff` |
| `b` | 边框颜色 | `b=#ddd` |
| `r` | 圆角 | `r=8` |
| `size` | 字体大小 | `size=16` |
| `bold` | 加粗 | `bold` |
| `note` | 功能说明 | `note="点击提交表单"` |

### 表格行属性

**表格行（`#`）支持以下属性：**
- `bg` – 整行的背景色
- `c`、`size`、`bold`、`italic`、`align` – 行内所有单元格的文字样式默认值

**重要说明：**
- `note` 属性**不支持**在表格行上使用
- 行级属性作为该行所有单元格的默认值
- 单元格可以单独设置属性，覆盖行级属性

**示例：**
```solarwire
## @(100,50) w=500 border=1
  # bg=#4CAF50 c=white bold      // 表头行：绿色背景，白色加粗文字
    "ID"
    "姓名"
    "操作"
  # bg=#f5f5f5                    // 数据行：浅灰色背景
    "1"
    "张三" c=#333                 // 覆盖：深色文字
    ["编辑"] ["删除"]
```

### 坐标系统

- **绝对坐标**：`@(100,50)` - 相对于画布左上角
- **相对坐标**：`@(0,+50)` - 相对于上一个元素向下偏移 50px

### SolarWire 代码块要求

**每个代码块必须：**

1. 以文档级声明开头
2. 第一个元素是容器矩形 `[] @(0,0) w=... h=... bg=#fff`
3. 所有元素有坐标
4. 使用注释 `//` 分隔不同区域
5. 布局贴近实际页面

**完整示例：**
```solarwire
!title="登录页"
!c=#333
!size=14
!bg=#f5f7fa
!r=8

// 容器矩形 - Web端1440宽度
[] @(0,0) w=1440 h=900 bg=#fff

// 页面标题
"企业管理系统" @(570,80) size=28 bold
"登录" @(680,130) size=18 c=#666

// 登录表单
"用户名" @(570,200)
["请输入用户名"] @(570,225) w=300 h=44 bg=#fff b=#ddd note="输入框，支持手机号或邮箱登录。
- 自动去除首尾空格
- 格式校验：手机号11位数字，邮箱需含@和域名
- 错误提示：格式错误时下方显示「请输入正确的手机号或邮箱」
- 最大长度：50字符"

"密码" @(570,290)
["请输入密码"] @(570,315) w=300 h=44 bg=#fff b=#ddd note="密码输入框。
- 密码显示为圆点
```

## 工作流程

### 1. 需求收集

```
好的！我来帮你创建 PRD 和线框图。请告诉我：

1. 这是什么类型的应用？
   - 📱 移动端 App
   - 💻 客户端 Web  
   - ⚙️ 后台管理系统

2. 需要哪些页面/功能？

3. 有什么特殊要求？
```

### 2. 需求确认

```
我理解的需求如下：

**场景**：[场景类型]
**页面**：[页面列表]
**核心功能**：
- [功能1]
- [功能2]

**布局建议**：
- [布局描述]

确认后我开始生成，需要调整吗？
```

### 3. 生成输出

**重要：本 Skill 的主要目的是生成完整的 PRD，线框图是 PRD 的一部分。**

生成内容：
- **MD 文档** → `{{outputDir}}/solarwire-[项目名].md`

**预览方式**：使用 VSCode SolarWire 插件实时预览。

### 4. 迭代修改

支持用户对已生成的线框图进行修改：
- 调整布局
- 添加/删除元素
- 修改 note 说明
- 添加新页面/弹窗/状态

## 重要提醒

1. **坐标必须完整** - 每个元素都要有 `@(x,y)`
2. **属性不用方括号** - 直接写 `w=100 h=40`
3. **Note 要详细但不写视觉/技术细节** - 功能行为、成功失败场景、输入限制、错误提示
4. **元素选择要合理** - 按钮用矩形，标签用文本，图标才用占位符
5. **布局贴近实际** - 线框图要能反映真实页面结构
6. **弹窗/状态/标签页分离** - 每个独立视图单独一个代码块
7. **容器矩形必须有** - 每个页面第一个元素是白色背景容器
